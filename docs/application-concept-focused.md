# アプリケーションコンセプト（絞り込み版）

## コンセプト名
Cross-Repo Review Console

## 一言でいうと
複数リポジトリに散らばる Issue / Pull Request を「1つの受信トレイ」で高速に処理するための、メンテナ向けデスクトップクライアント。

## 解決したい課題
- リポジトリごとに画面を行き来するため、Issue/PR の見落としが起きる
- close、approve、review などの定型操作に時間がかかる
- 変更差分の比較を開くまでにステップが多く、レビュー開始が遅れる

## 対象ユーザー
- 複数リポジトリを同時に管理する maintainer / reviewer
- 毎日 PR・Issue をトリアージする開発リーダー

## プロダクト原則
- 横断性: リポジトリ単位ではなく「作業キュー単位」で情報を扱う
- 速度: 一覧→判断→操作をキーボード中心で最短遷移にする
- 文脈維持: 一覧、詳細、差分、操作結果を同じ作業面で完結させる
- 安全性: 破壊的操作は確認可能にしつつ、通常操作は極力ノンブロッキングにする

## コア機能（このアプリの主機能）
1. 横断検索 / 一覧
- 複数 repo の Issue / PR を統合表示
- 条件: owner/repo, state, label, assignee, reviewer, draft, updated_at
- 保存ビュー（例: 「自分宛レビュー待ち」「7日以上更新なし」）

2. 高速アクション
- Issue: close / reopen / assignee / label 更新
- PR: approve / request changes / comment / close / reopen / merge
- 一覧上のクイック操作 + 一括処理（安全ガード付き）

3. レビュー体験
- PR 詳細から差分に1アクションで遷移
- ファイルツリー + unified diff の2ペイン表示
- コメント位置へのジャンプ、未読/未対応スレッドの可視化

4. 作業キュー管理
- 「今やるべき順」に並ぶ優先度ビュー
- SLA 風ルール（例: レビュー依頼から24時間超過）でハイライト
- 実行履歴（何をいつ操作したか）を追跡

## 非対象（意図的にやらないこと）
- GitHub 全機能の網羅（Actions, Releases, Projects, Settings 全般）
- 組織管理や課金・監査など repo 境界を超える管理機能
- IDE 相当のコード編集機能

## UX 方針
- 1画面3レーン構成
- 左: 検索条件 / 保存ビュー
- 中央: 横断一覧（Issue/PR 混在可）
- 右: 詳細 + 差分 + アクション
- 主要操作はショートカット提供（例: `A` approve, `C` close, `R` review コメント）

## 差別化ポイント
- GitHub Web より「横断キュー処理」に特化
- CLI より「比較・レビュー文脈保持」に強い
- 単なるビューアではなく、判断と操作を同じ導線で完結

## 成功指標（KPI）
- 初動時間: 一覧表示から最初の操作までの時間を 30 秒未満
- 処理速度: 1セッションあたりの triage 件数
- 取りこぼし率: 期限超過 PR/Issue の件数
- 操作効率: close/approve/review の平均操作ステップ数

## MVP スコープ（最初のリリース）
- 横断一覧（Issue/PR）
- 保存ビュー（最低3種）
- PR approve / request changes / comment / close
- Issue close / reopen / comment
- 2ペイン差分ビュー（ファイル一覧 + unified diff）
- 基本ショートカットと操作履歴

## 将来拡張（MVP 後）
- ルールベース自動振り分け（ラベル/担当者）
- チーム別ダッシュボード
- 通知連携（Slack など）

## メッセージ（対外説明用）
GitHub の全機能を置き換えるのではなく、
「複数リポジトリの Issue / PR を、速く・漏れなく・迷わず処理する」ことに集中したレビュー運用コンソール。
